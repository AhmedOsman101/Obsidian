Here is the reorganized list of normalization exercises. I have paired each requirement with its corresponding source table to make the context immediately clear.

## Normalization Exercises

---

### **1. Requirement: Normalize Table 1 to 1NF**

_Scenario: A department table listing multiple students in a single cell._

| **Department_Name** | **Student_Name**  |
| ------------------- | ----------------- |
| BE-IT               | Rasmus, Jens, Kim |
| Multimedia Design   | Sara              |

---

### **2. Requirement: Normalize Table 2 to 1NF**

_Scenario: A product table where prices are stored in repeating columns._

| **Product_ID** | **Product_Name** | **Price_1** | **Price_2** |
| -------------- | ---------------- | ----------- | ----------- |
| 200            | Table            | 708.67      | 800.00      |

---

### **3. Requirement: Normalize Table 3 to 1NF**

_Scenario: A student enrollment table with repeating course columns._

| **Student_ID** | **Student_Name** | **Course1** | **Course2** |
| -------------- | ---------------- | ----------- | ----------- |
| S101           | John Smith       | Math        | Science     |

### **4. Requirement: Normalize Table 4 to 2NF**

_Scenario: Primary key is `Lecturer_ID`, but it contains department information that should be separate._

| --------------- | ------------------ | ----------------- | ------------------- |
| 100             | Jesper Frederiksen | 022               | BEIT                |

### **5. Requirement: Normalize Table 5 to 2NF**

_Scenario: Composite key `{S#, P#}` where `STATUS` and `CITY` only depend on `S#`._

| **S#** | **STATUS** | **CITY** | **P#** | **QTY** |
| ------ | ---------- | -------- | ------ | ------- |
| S1     | 20         | London   | P1     | 300     |

### **6. Requirement: Normalize Table 6 to 2NF**

### **Answer Tables (Normalized Results)**

**Ans 1 (1NF):** Eliminate repeating values by creating a record for each student.

| Department_Name | Student_Name |
| :-------------- | :----------- |
| BE-IT           | Rasmus       |
| BE-IT           | Jens         |
| BE-IT           | Kim          |

**Ans 2 (1NF):** Store each price as a separate record.

| Product_ID | Product_Name | Price  |
| :--------- | :----------- | :----- |
| 200        | Table        | 708.67 |
| 200        | Table        | 800.00 |

**Ans 3 (1NF):** Create one record per student-course pair.

| Student_ID | Student_Name | Course  |
| :--------- | :----------- | :------ |
| S101       | John Smith   | Math    |
| S101       | John Smith   | Science |

**Ans 4 (2NF):** Decompose into **Lecturer Table** (ID, Name, Dept_ID) and **Department Table** (Dept_ID, Dept_Name).

**Ans 5 (2NF):** Decompose into **Supplier Table** (S#, Status, City) and **Shipment Table** (S#, P#, QTY).

**Ans 6 (2NF):** Decompose into **Diagnosis Table** (Code, Description) and **Service Table** (ProviderID, Code, Date).

**Ans 7 (3NF):** Decompose to remove transitive dependency: **Dept Table** (ID, Name, City) and **City Table** (City, Zip_Code).

**Ans 8 (3NF):** Decompose into **Customer Table** (ID, AccNo, BankCode) and **Bank Table** (BankCode, Bank).

**Ans 9 (3NF):** Decompose into **Patient Table** (CPR, Name, Insurance_Num) and **Insurance Table** (Insurance_Num, Insurance_Name).

**Ans 10 (3NF):** Decompose into **Dept Address** (ID, Street, City) and **Zip Lookup** (City, Zip_Code).
