# Task 1: Design a Company Network Using Packet Tracer

You are required to use Cisco Packet Tracer to design and simulate a company network that supports 50 computers.

## 🎯 Objectives:

Design a functional, scalable, and efficient network for a medium-sized company.

## 📝 Requirements:

1. Network Design

- Create a complete network topology in Packet Tracer.
- Connect 50 PCs to the network.
- Ensure all devices can communicate properly.
- Assign appropriate IP addressing.

2. Topology Selection

- Clearly state which topology you used (Star, Extended Star, Bus, Ring, Hybrid, etc.).
- Explain:
  - Why you selected this topology.
  - Its advantages for a 50-computer company.
  - Any limitations.

3. Physical Components

- List and justify all physical components used, such as:
  - Routers
  - Switches
  - Hubs (if any)
  - Cables (type: straight-through, crossover, fiber, etc.)
  - Servers (if included)
- For each component, explain:
  - Why you chose it.
  - Its role in the network.

4. Device Quantity Justification

- Explain how you decided:
  - How many routers are needed.
  - How many switches are required.
  - Whether hubs are necessary or not.
- Show your calculation or reasoning (e.g., port count, segmentation, performance, scalability).

5. Network Planning Considerations

- Briefly explain:
  - How you ensured scalability for future growth.
  - How you avoided network congestion.
  - Basic security considerations (VLANs, firewall, etc., if implemented).

## 📤 Deliverables:

- Packet Tracer (.pkt) file.
- A short report (2–3 pages) explaining:
  - Topology choice
  - Components used
  - Device quantity justification
  - Design decisions

## ⭐ Evaluation Criteria:

- Correct functionality
- Logical topology choice
- Proper device calculation
- Clear technical explanation
- Professional presentation
