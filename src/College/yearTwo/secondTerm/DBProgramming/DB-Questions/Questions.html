Database Programming — Revision Questions

SQL Fundamentals

1- What is a relational database?

A database that stores information in tabular form (rows and columns), where relationships between tables are defined via keys (e.g., Product table linked to Color table via Color ID).

2- What are the five categories of SQL commands?
CategoryPurposeExamples
DDLDesign DB structureCREATE, ALTER, DROP
DQLRetrieve dataSELECT
DMLWrite/modify recordsINSERT, UPDATE, DELETE, TRUNCATE
DCLManage user accessGRANT, REVOKE
TCLTransaction controlCOMMIT, ROLLBACK, SAVEPOINT
3- What are the components of a SQL system?
  1. Table — rows and columns, the basic storage unit
  2. Field / Attribute — a column in a table
  3. Record / Row — an individual entry
  4. Column — vertical entity for a specific field
  5. NULL — a field with no value (different from zero or spaces)
4- What is the difference between SQL and MySQL?

SQL is a standard query language for relational databases. MySQL is an open-source RDBMS that uses SQL queries. SQL is the language; MySQL is one implementation of it.

5- List three SQL numeric data types and three string data types.

Numeric: INT, DECIMAL, FLOAT.
String: VARCHAR, CHAR, TEXT.

6- Write SQL to create a database called SchoolDB and then drop it. 
CREATE DATABASE SchoolDB;
DROP DATABASE SchoolDB;

Both require admin privilege.

7- What is the difference between TRUNCATE and DELETE?

TRUNCATE deletes all rows but keeps the table structure (DDL — cannot use WHERE). DELETE is DML — can use WHERE to delete specific rows; without WHERE it deletes all rows but is slower than TRUNCATE.

8- What does SHOW DATABASES and DESCRIBE Students do?

SHOW DATABASES lists all databases on the server. DESCRIBE Students (or DESC Students) displays the structure (columns, types, constraints) of the Students table.

Constraints

9- What is the difference between column-level and table-level constraints?

Column-level constraints apply to a single column (e.g., Age INT NOT NULL). Table-level constraints can involve multiple columns (e.g., CONSTRAINT PK_Person PRIMARY KEY (ID, LastName)).

10- Explain the CHECK constraint with an example.

CHECK limits the values that can go into a column. Example:

CREATE TABLE Employees (
  Salary DECIMAL CHECK (Salary > 0),
  Department VARCHAR(50) CHECK (Department IN ('IT', 'HR', 'Sales'))
);
11- What is a composite primary key? Give an example.

A PRIMARY KEY made up of two or more columns. Example:

CREATE TABLE Enrollment (
  student_id INT NOT NULL,
  course_id  INT NOT NULL,
  PRIMARY KEY (student_id, course_id)
);
12- How do you drop a PRIMARY KEY and a UNIQUE constraint? 
ALTER TABLE Persons DROP PRIMARY KEY;
ALTER TABLE Persons DROP INDEX UC_Person;  -- for UNIQUE
13- Write SQL to add a FOREIGN KEY on Students.dept_id referencing Departments.dept_id. 
ALTER TABLE Students
ADD CONSTRAINT fk_student_dept
FOREIGN KEY (dept_id) REFERENCES Departments(dept_id);
14- What is the difference between WHERE and HAVING?

WHERE filters rows before grouping. HAVING filters groups after GROUP BY. You cannot use aggregate functions in WHERE.

15- What does COUNT(*) return vs COUNT(column)?

COUNT(*) counts all rows including NULLs. COUNT(column) counts only non-NULL values in that column.

16- What is the difference between IFNULL and NULLIF?

IFNULL(x, alt) returns alt if x is NULL, otherwise x. NULLIF(a, b) returns NULL if a = b, otherwise returns a.

17- What does DATE_ADD('2026-06-04', INTERVAL 1 MONTH) return?

2026-07-04. Related functions: NOW(), CURDATE(), CURTIME(), DATE_SUB().

18- What are the three parameter types in a stored procedure?
TypeDescription
INInput parameter (default)
OUTReturns a value to the caller
INOUTBoth input and output
19- Why should indexes be used selectively?

Indexes speed up SELECT queries but slow down INSERT, UPDATE, and DELETE (the index must be updated on every write). Only index columns that are frequently searched.

20- What are the two goals of database normalization?
  1. Eliminate redundant data (same data stored in multiple tables)
  2. Ensure data dependencies make sense (only related data in the same table)
21- List the four types of NoSQL databases.

Key-Value (Redis), Column-Oriented (Cassandra), Document-Oriented (MongoDB), Graph (Neo4j).

22- What is SQL injection?

A cyberattack that inserts malicious SQL queries into input fields (e.g., login forms) to retrieve, modify, or corrupt data. Prevent it with parameterized queries/prepared statements.

Stored Procedures & Triggers

82- Create a procedure to return the highest salary in a given department. 
DELIMITER $$

CREATE PROCEDURE GetHighestSalary(IN p_dept_id INT)
BEGIN
  SELECT d.dept_name, MAX(e.salary) AS highest_salary
  FROM Employees e
  JOIN Departments d ON e.dept_id = d.dept_id
  WHERE e.dept_id = p_dept_id;
END$$

DELIMITER ;

CALL GetHighestSalary(1);
83- Create a procedure to increase salary by a given percentage for a department. 
DELIMITER $$

CREATE PROCEDURE IncreaseSalary(IN p_dept_id INT, IN p_percent DECIMAL(5,2))
BEGIN
  UPDATE Employees
  SET salary = salary * (1 + p_percent / 100)
  WHERE dept_id = p_dept_id;
END$$

DELIMITER ;

CALL IncreaseSalary(2, 15);  -- 15% raise for dept 2
84- Create a BEFORE UPDATE trigger to prevent salary decrease. 
DELIMITER $$

CREATE TRIGGER prevent_salary_decrease
BEFORE UPDATE ON Employees
FOR EACH ROW
BEGIN
  IF NEW.salary < OLD.salary THEN
    SIGNAL SQLSTATE '45000'
      SET MESSAGE_TEXT = 'Error: Salary cannot be decreased.';
  END IF;
END$$

DELIMITER ;
85- Create an AFTER DELETE trigger to log deleted employees. 
CREATE TABLE IF NOT EXISTS Deleted_Employees_Log (
  log_id     INT PRIMARY KEY AUTO_INCREMENT,
  emp_id     INT,
  name       VARCHAR(100),
  salary     DECIMAL(10, 2),
  dept_id    INT,
  deleted_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

DELIMITER $$

CREATE TRIGGER log_deleted_employee
AFTER DELETE ON Employees
FOR EACH ROW
BEGIN
  INSERT INTO Deleted_Employees_Log (emp_id, name, salary, dept_id, deleted_at)
  VALUES (OLD.emp_id, OLD.name, OLD.salary, OLD.dept_id, NOW());
END$$

DELIMITER ;

Comprehensive

86- Create a query returning each customer's name, total order count, and "VIP" (>5 orders) or "Regular" status. 
SELECT
  c.CustomerName,
  COUNT(o.OrderID) AS OrderCount,
  IF(COUNT(o.OrderID) > 5, 'VIP', 'Regular') AS Status
FROM Customers c
LEFT JOIN Orders o ON c.CustomerID = o.CustomerID
GROUP BY c.CustomerID, c.CustomerName;
87- Given Students(id, name) and Enrollments(student_id, course, grade), show each student's average grade, only for students with avg > 75. 
SELECT s.name, AVG(e.grade) AS AvgGrade
FROM Students s
INNER JOIN Enrollments e ON s.id = e.student_id
GROUP BY s.id, s.name
HAVING AVG(e.grade) > 75
ORDER BY AvgGrade DESC;
